# TestPilot ‚Äî Claude Code Instructions

## What This Project Is

TestPilot is a Kotlin Multiplatform (KMM) SDK for AI-driven mobile app testing using vision (screenshots). It supports two modes: `./testpilot analyze` (exploratory UX analysis ‚Üí HTML report) and `./testpilot test` (deterministic pass/fail ‚Üí CI exit codes). Supports iOS (XCTest) and Android (UIAutomator).

## Tech Stack

- **Core SDK**: Kotlin Multiplatform 1.8, Gradle (JDK 11+)
- **AI client**: `com.aallam.openai:openai-client:3.1.1` (aallam)
- **HTTP**: Ktor 2.2.4
- **iOS wrapper**: Swift + SPM, consumes KMM-built XCFramework via cinterop
- **Android**: UIAutomator 2.2.0, min SDK 29

## Project Structure

```
sdk/testpilot/src/
  commonMain/   ‚Üê Platform-agnostic logic: Analyst, TestAnalyst, prompts, Config
  androidMain/  ‚Üê UIAutomator-backed snapshot + actor
  iosMain/      ‚Üê XCTest cinterop + .def files, iOS snapshot + actor, CachingAIClient
swift-wrapper/  ‚Üê Thin Swift API on top of KMM XCFramework
scripts/        ‚Üê build_ios_sdk.sh
mac-app/        ‚Üê macOS companion app (SwiftUI)
```

## Build Commands

```bash
# Build iOS XCFramework (run from repo root)
scripts/build_ios_sdk.sh

# Build Android AAR
cd sdk && ./gradlew testpilot:assembleDebug

# Build release XCFramework
scripts/build_ios_sdk.sh testpilot:assembleTestPilotSharedReleaseXCFramework
```

## Key Files

## Code Conventions

- Kotlin: standard KMM patterns, `expect`/`actual` for platform abstractions
- Swift: extensions on `XCTestCase`, `@MainActor` async
- Commit style: `type(scope): description` (e.g. `feat(KMM): add persistence api`)
- No tests directory in this repo ‚Äî TestPilot IS the test framework

## Important Notes

- The `.def` files in `iosMain/` are generated by `build_ios_sdk.sh` from `.templ` files ‚Äî do not edit `.def` files directly
- `Package.swift` references the XCFramework at a build output path; it won't resolve until the iOS SDK is built
- Config requires at minimum an `apiKey`; all other fields have defaults (maxTokens=200, maxSteps=40, temperature=0.0)
- Test mode stdout markers: `TESTPILOT_STEP: <message>`, `TESTPILOT_STEP: (cached) <message>`, `TESTPILOT_RESULT: PASS <reason>`, `TESTPILOT_RESULT: FAIL <reason>`
- `CachingAIClient` cache path: `NSCachesDirectory/testpilot-cache/` ‚Äî scoped to the XCTest runner container, persists across separate runs of the same test target
